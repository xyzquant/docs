The oracle price serves two critical functions on trade[XYZ]: it is the reference price used to calculate the funding rate, and it is a direct input to the mark price calculation. The key unlock for equity perpetuals is extending the oracle to operate 24/7, well beyond the trading hours of the underlying asset. To achieve this, XYZ prioritizes external pricing and employs a robust internal mechanism whenever external data is unavailable.

## External Pricing

When external markets are open, the Relayer aggregates price data from a robust set of venues, markets, and institutional data providers. The externally-derived fair price is transmitted as the oracle price in each Relayer update. Instrument-specific details on how external prices are derived can be found in each asset's documentation.

## Internal Pricing

When external inputs are unavailable — such as overnight, on weekends, or during market holidays — the oracle advances via a continuous-time exponentially weighted moving average (EMA). This mechanism incrementally adjusts the previous oracle price by a fraction of the **Impact Price Difference (IPD)**.

With oracle price `S`, the IPD is defined as:

```text
IPD = max(P_impactBid - S, 0) - max(S - P_impactAsk, 0)
```

Where `P_impactBid` and `P_impactAsk` are the average execution prices to trade a configured impact notional amount on the bid and ask sides of the order book, respectively. If insufficient depth exists on either side, that side's contribution is set to zero.

The oracle is robust to irregular updates and market halts. Using a time constant `τ = 30 minutes`, it updates according to:

```text
S_t = β_t · S_{t-} + (1 - β_t) · x_t

β_t = exp(-Δt* / τ)

x_t = S_{t-} + IPD_t
```

Where:

- `Δt* = min(Δt, c · τ)` caps the effective elapsed time per update
- `Δt = t - t_prev` is the actual time since the last update
- `c = 0.1`, so the maximum weight applied in any single update is `1 - exp(-0.1) ≈ 9.5%`

This cap prevents any single large time gap from causing an outsized jump in the oracle price.

### Resuming External Pricing

When external data becomes unavailable, the internal mechanism initializes from the last available external price, ensuring continuity. When external inputs resume, the oracle reverts to the externally derived price on the next tick.

<Note>
  The formulas above describe the general form of the continuous-time EMA used for the oracle. Component 2 of the mark price uses the same EMA structure but samples the basis — the difference between the perpetual's mid-price and the oracle price — with a shorter time constant of `τ = 150 seconds`.
</Note>
